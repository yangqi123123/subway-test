## 二、模块级详细业务流程（逐点可执行）

### 1\. 全景地图（GIS）详细流程

```Plain Text
1. 左侧图层栏加载
   图层：站点、项目、人员、告警点、巡查区域、地铁线路、应急资源分布
   工具：坐标拾取、测量距离、测量面积

2. 线路渲染流程
   线路管理 → 关联站点 → 关联区间
   用户点击线路图层 → 地图渲染该线路所有站点、区间、项目、巡查区域

3. 点位数据来源
   站点 → 后台录入经纬度 → 地图显示点位
   项目 → 后台录入经纬度 → 地图显示点位
   人员 → 第三方平台实时轨迹 → 地图动态刷新
   告警点 → 第三方无人机推送 → 地图显示告警图标
   巡查区域 → 人工巡检创建记录 → 已巡查=绿色，待巡查=橙色
   已上传照片工地 → 显示水滴图标 → 鼠标悬浮展示最新照片
   应急资源 → 应急预案绑定部门 → 渲染该部门下所有应急人员点位

4. 交互流程
   点击任意图标 → 弹出详情弹窗
   地图工具使用 → 实时显示坐标/距离/面积
```

### 2\. 虚拟座舱详细流程

```Plain Text
1. 进入全景地图 → 点击底部【虚拟座舱】
2. 选择机场 → 加载该机场下无人机列表
3. 选择无人机 → 检查起飞条件
   - 航线空域有效性
4. 满足条件 → 点击【起飞】
   → 显示机场摄像头实时视频流
   → 无人机上升阶段：画面不变，地图位置不变
   → 完全升空 → 自动切换为实时地图画面
5. 不满足条件 → 弹窗提示：不满足起飞条件，无法起飞
6. 飞行中面板展示
   电量、风速、高度、速度、温度、位置、飞行时间
7. 操控流程
   解锁手动操控 → 接管飞行
   智能返航 → 自动返回机场
   急停 → 立即悬停
```

### 3\. 飞行计划详细流程（核心审批链路）

```Plain Text
1. 飞手端流程
   登录 → 进入飞行计划 → 【新增计划】
   填写：计划名称、线路、起降机场、无人机、航线、飞行策略、执行时间
   选择航线 → 提交审批 → 状态变为：待审批

2. 管理员审批流程
   收到待办 → 查看计划详情、航线空域有效性
   操作：通过 / 驳回
   通过 → 计划状态：已通过，可执行
   驳回 → 备注原因 → 退回飞手修改

3. 执行流程
   飞手进入虚拟座舱 → 选择已通过计划
   系统自动校验：
   - 航线是否存在
   - 空域是否在有效期
   - 无人机是否可用
 
   校验通过 → 无人机按预设航线自动飞行
   飞行完成 → 自动返航 → 降落
   飞行结束 → 本平台自动生成飞行日志（不同步第三方）

4. 异常流程
   空域无效 → 禁止起飞
   无人机离线 → 禁止起飞
   飞行中断 → 记录异常 → 日志标记异常状态
```

### 4\. 航线管理详细流程

```Plain Text
1. 航线数据来源
   第三方无人机平台 → 自动同步航线 → 进入本平台航线库

2. 空域申请流程
   选择航线 → 前往指定官网提交空域申请
   填写：申请单位、时间、范围、无人机信息
   审批通过 → 本平台手动标记：空域有效
   空域过期 → 自动标记：空域无效

3. 关联逻辑
   航线必须关联线路、机场
   飞行计划创建时 → 必须选择一条有效航线
```

### 5\. 告警信息详细流程

```Plain Text
1. 告警接入
   第三方无人机平台 → 实时推送告警 → 存入本平台告警库
   告警类型：病害、施工、异常振动、侵入、其他

2. 地图展示
   GIS地图渲染告警点位 → 红色图标闪烁

3. 告警详情页展示内容
   告警位置、时间、来源、等级、描述
   无人机实拍图片、视频
   处警记录、处理状态、复核结果

4. 处置流程
   值班人员查看告警 → 现场核查
   填写：是否误报、复核情况、现场照片
   更新状态：已处理 / 误报 / 待处理
   记录存入历史告警列表 → 支持筛选、导出
```

### 6\. 态势感知详细流程

```Plain Text
1. 数据来源
   第三方无人机平台 → 告警数据 + 线路监测数据

2. 页面结构
   左侧GIS地图 → 展示告警点位
   左侧告警列表 → 最新告警排序
   右侧瀑布图（左线 + 右线）

3. 可视化规则
   正常 → 蓝色
   预警 → 黄色
   险情 → 橙色 / 红色
   对应区间自动变色

4. 交互
   点击告警列表项 → 地图定位
   点击异常区间 → 进入专家工具
```

### 7\. 专家工具详细流程

```Plain Text
1. 进入方式：态势感知点击区间进入

2. 页面布局
   左侧：GIS地图 → 定位告警区间
   中上：瀑布图
     横轴：监测点位位置
     纵轴：时间
     蓝色=正常 黄色/红色=异常
   右上：频谱图 → 数据波动热力展示
   底部：卡片组

3. 底部卡片固定内容
   卡片1：最新告警详情（位置、时间、来源、项目）
   卡片2：最新告警复核情况（是否误报、描述、照片）
   卡片3：历史告警信息1
   卡片4：历史告警信息2
   卡片5：历史告警信息3
   卡片6：当前位置备注（可编辑）

4. 数据来源
   全部来自第三方无人机平台
```

### 8\. 巡检记录详细流程

```Plain Text
1. 病害巡检流程
   选择线路 → 区间 → 站点
   填写病害描述、位置、等级
   上传照片/视频 → 提交

2. 夜班作业巡检
   新建夜班记录 → 填写作业内容、单位、时间
   上传资料 → 提交 → 工班确认

3. 人工巡检
   选择项目 → 填写巡检结果
   提交 → 标记该区域为【已巡查】→ 地图变绿色

4. 无人机巡检
   第三方平台自动生成 → 同步本平台 → 生成报告
   支持查看、导出、复核

所有巡检记录 → 进入巡检质量统计 → 用于打分、排行、频次统计
```

### 9\. 地铁路网管理流程（线路 \- 站点 \- 区间）

```Plain Text
1. 线路管理
   后台新增线路 → 填写名称、标志色、起止站

2. 站点管理
   新增站点 → 选择所属线路 → 填写经纬度、里程、长度
   保存 → 同步到GIS地图

3. 区间管理
   新增区间 → 选择所属线路 → 上下行
   关联前后站点 → 填写里程
   保存 → 形成完整线路

4. 关联关系
   线路 ← 一对多 → 站点
   线路 ← 一对多 → 区间
   区间 ← 关联 → 两个站点
```

### 10\. 应急预案模块详细流程

```Plain Text
1. 基础数据
   应急人员 → 新增 → 绑定部门
   应急仓库 → 新增 → 绑定部门 → 添加应急物资
   应急预案 → 新增 → 绑定部门

2. GIS渲染流程
   打开全景地图 → 选择应急资源图层
   选择预案 → 根据部门 → 加载该部门下所有应急人员点位
   地图显示人员位置、联系方式、岗位

3. 使用场景
   告警触发 → 启动预案 → 调度对应部门应急人员
```

### 11\. 设备管理流程（机场 / 无人机 / 维修 / 日志）

```Plain Text
1. 机场设备
   本平台新增 → 填写信息 → 同步第三方无人机平台

2. 无人机设备
   本平台新增 → 绑定机场 → 同步第三方无人机平台

3. 维修记录
   本平台新增 → 关联设备 → 填写内容、配件、时间
   状态：待处理/进行中/已完成
   【不同步第三方】

4. 飞行日志
   飞行任务结束 → 自动生成
   内容：任务信息、起降时间、电量、高度、轨迹、事件
   【不同步第三方】
```

### 12\. AI 识别详细流程

```Plain Text
1. 预警接入
   第三方无人机平台 → AI识别风险 → 推送预警
   风险类型：侵入、病害、异常施工、安全隐患

2. 前端展示
   地图显示风险点位
   支持：风险图片 ↔ 风险视频 实时切换

3. 审批流程（工作流）
   查看预警 → 填写复核意见、处理建议
   提交审批 → 多级审批
   审批通过 → 预警闭环
   审批不通过 → 退回修改

4. 记录留存
   预警记录、图片、视频、审批意见 → 永久保存
```
